## 오늘의 목표 | Today's Goal

어제(Day 039) 결론: 임베딩 기반 RAG 검색은 구조가 거의 동일한 Day 로그들 사이에서 정확한 날짜를 못 찾는다. 문서에 힌트 문장을 추가해도 실패했다 — 이건 설정이나 프롬프트로 풀리는 문제가 아니라 기술 자체의 한계다.

**오늘은 AI 검색에 맡기지 않는다.** 날짜 질문이 들어오면:
1. 정규식으로 질문에서 날짜(`YYYY-MM-DD`) 또는 "Day NNN" 패턴을 추출
2. `Daily_Log/` 폴더의 파일명을 미리 스캔해서 만든 **날짜 ↔ 파일명 딕셔너리**에서 정확히 찾기
3. 찾은 파일을 직접 읽어서 그 내용을 컨텍스트로 Ollama(cravveo:v4)에 전달, 답변 생성

이건 AI가 아니라 그냥 파이썬 딕셔너리 조회라서, 파일이 존재하기만 하면 100% 정확하다.

재학습 없음, RAG 설정 변경 없음 — 별도의 Python 스크립트로 새 기능을 만든다.
No retraining, no RAG config changes — building a new standalone Python script.

## 작업 순서 | Steps

### Step 1. 날짜 ↔ Day 번호 매핑 딕셔너리 만들기
- `Daily_Log/` 폴더의 파일명을 스캔 (`2026-06-27_Day032_Log.md` 형태)
- 정규식으로 날짜와 Day 번호 추출 → `{"2026-06-27": "Day032", ...}` 딕셔너리 생성
- 예외 케이스 처리: `Day020+021`(복합 Day), `Special`(Day 번호 없음)

### Step 2. 질문에서 날짜 추출하는 함수
- 사용자 질문("2026-06-27일에 뭐했어?")에서 정규식으로 날짜 패턴 추출
- "Day 032에 뭐했어?" 같은 Day 번호 질문도 인식하도록 처리

### Step 3. 파일 읽기 + Ollama 호출
- 딕셔너리에서 찾은 파일을 직접 읽기 (`Daily_Log/2026-06-27_Day032_Log.md`)
- 파일 내용을 컨텍스트로 넣어서 Ollama API(`cravveo:v4`)에 질문 + 컨텍스트 전달
- 답변 받아서 출력

### Step 4. 검증 테스트
- Day 039에서 썼던 10개 질문 그대로 재사용
- 이번엔 O/X 대신 "정확히 그 파일 내용을 찾았는가"만 확인 (100% 나와야 정상)
