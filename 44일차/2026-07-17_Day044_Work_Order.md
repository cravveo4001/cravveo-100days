## 작업 순서 | Steps

### Step 1. 정체성 데이터 60~80개로 확장
- 기존 20개 사실(`10일차/cravveo_briefing_dataset.jsonl`) 유지
- 각 사실마다 표현을 3~4가지로 다르게 써서 추가
- 결과를 새 파일(`44일차/cravveo_identity_expanded.jsonl`)로 저장

### Step 2. Colab에서 큰 모델 로드 테스트
- `44일차/Day044_BigModel_Test.ipynb` 작성 — Llama 3.1 8B(또는 Qwen 2.5 7B)를 4비트로 로드만 해보고 GPU 메모리 사용량 확인
- 로드 성공하면 간단한 질문 1~2개로 기본 동작 확인 (파인튜닝 없이 원본 모델 상태로)

### Step 3. 결과 정리
- 데이터 개수, 로드 성공 여부, 메모리 사용량 기록
- 다음 재학습(v7)에 8B를 쓸지 3B를 유지할지 결정 근거 마련
