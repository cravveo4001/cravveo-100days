## 🔵 Phase 4: AI 참모와 옵시디언 금고의 완전 오프라인 RAG 합체 (Day 061 ~ Day 080)

### Day 061: 로컬 RAG 지식 OS 설계도 작성
*   **미션**: 노트북 안의 수백 개의 메모들을 로컬 AI가 어떻게 참고하는지 파이썬 기반 RAG 시스템의 작동 모형을 구상합니다.

### Day 062: 파이썬으로 옵시디언 마크다운 리스트 확보
*   **미션**: 지정된 옵시디언 금고 폴더의 모든 `.md` 파일 경로를 파이썬의 `glob` 라이브러리로 수집하는 코드를 짭니다.

### Day 063: 텍스트 전처리(Clean text) 스크립트 작성
*   **미션**: 마크다운 특수문자, HTML 태그, 불필요한 줄바꿈 등을 말끔히 청소해 주는 파이썬 정규식 함수를 구현합니다.

### Day 064: 텍스트 청킹(Chunking) 유틸리티 작성
*   **미션**: 긴 글을 300자 내외로 쪼개어 AI가 이해하기 좋은 크기로 분류하는 파이썬 코드를 짭니다.

### Day 065: 로컬 단어 임베딩(Word Embedding)의 정체
*   **미션**: 텍스트 단어들을 다차원 공간의 좌표값(숫자)으로 바꾸어 컴퓨터가 이해하게 만드는 핵심 이론을 공부합니다.

### Day 066: sentence-transformers 라이브러리 설치
*   **미션**: 가볍고 성능 좋은 로컬 전용 한국어 임베딩 모델을 파이썬에 로드하고 단어를 벡터로 변환해 봅니다.

### Day 067: 로컬 벡터 DB 'ChromaDB' 설치 및 연동
*   **미션**: 리눅스 환경에 경량 벡터 데이터베이스 ChromaDB 패키지를 설치하고 내 노트를 DB에 담습니다.

### Day 068: 유사 문서 검색 쿼리 작성
*   **미션**: 쿼리 함수를 작성하여 검색 결과를 리스트로 받아냅니다.

### Day 069: Ollama API Chat 완성
*   **미션**: 검색된 로컬 노트를 프롬프트에 동봉하여 내 로컬 API 모델에 쏘아 보내는 최종 RAG 코드를 짭니다.

### Day 070: 오프라인 질문 검증 (인터넷 차단 시연)
*   **미션**: 랜선을 뽑은 상태에서 RAG 비서가 정확히 대답하는지 확인합니다.

### Day 071: RAG 답변 신뢰도(Hallucination 방지) 체크
*   **미션**: 모르는 질문을 던졌을 때 RAG 비서가 거짓말하지 않고 "관련 문서가 없습니다"라고 솔직하게 고백하도록 튜닝합니다.

### Day 072: 청크 오버랩(Overlap) 최적화 튜닝
*   **미션**: 글을 쪼갤 때 앞뒤 문맥이 끊기지 않도록 오버랩 비율을 조절하여 답변 품질을 높입니다.

### Day 073: 옵시디언 감시자(File Watcher) 기능 초안
*   **미션**: 새로운 일지가 추가되면 자동으로 벡터 DB에 주입해 주는 실시간 동기화 파이썬 코드를 작성합니다.

### Day 074: 경량 임베딩 모델 교체 테스트
*   **미션**: 허깅페이스에서 다른 한국어 모델을 가져와 답변 속도와 정확성이 얼마나 개선되는지 비교해 봅니다.

### Day 075: 옵시디언 캔버스(Canvas) 파일 파서 도전
*   **미션**: JSON 구조로 되어 있는 옵시디언 마인드맵 파일(.canvas)의 노드 정보를 RAG가 읽을 수 있도록 특수 파서를 코딩합니다.

### Day 076: 파일 인코딩 에러(UTF-8) 디버깅
*   **미션**: 가끔 리눅스 환경에서 윈도우 한글 인코딩 파일이 꼬여서 파이썬 코드가 죽는 문제를 예외 처리(try-except)로 우회합니다.

### Day 077: RAG 풀 스택 파이썬 제어 실습
*   **미션**: 복잡한 상용 툴을 쓰지 않고 내가 한 땀 한 땀 짠 순수 파이썬 코드로 전체 지식 흐름이 장악되는 쾌감을 영상으로 보여줍니다.

### Day 078: RAG 비서에게 "오늘 작전 요약서" 출력 명령
*   **미션**: 금고의 일정 정보를 탐색해 오늘 반드시 해결해야 할 미션 우선순위를 똑똑하게 요약받는 모습을 시연합니다.

### Day 079: 시스템 메모리 과부하 모니터링
*   **미션**: RAG 시스템을 구동해 놓고 시스템 자원 사용량이 안정적으로 유지되는지 메모리 누수 테스트를 거칩니다.

### Day 080: Phase 4 성공 마무리 토크
*   **미션**: 지능(파인튜닝)과 지식(RAG)이 하나로 합쳐진 지식 OS의 완성을 선언합니다.

*   **미션**: AI가 직접 사령관님에게 오늘의 작전 브리핑을 음성으로 전달해 주는 오프라인 스피커 시스템을 구상합니다.

### Day 089: 리눅스용 pyttsx3 및 Coqui TTS 셋업

### Day 090: 대망의 첫 로컬 음성 작전 브리핑 가동
*   **미션**: "사령관님, 좋은 아침입니다."로 시작하는 100% 로컬 오프라인 스피커 기능 시연을 성공시키고 감상합니다.

### Day 091: 옵시디언 감시 알림 에이전트 코딩
*   **미션**: 작업 중 로그 파일에 "Error"라는 글자가 탐지되면 즉시 리눅스 시스템 알림창(Notify)을 띄우는 자동화 감시자를 붙입니다.

### Day 092: 파이썬 스크립트 CLI 도구 패키징
*   **미션**: 터미널에 `cravveo summary` 혹은 `cravveo email`이라고 치면 즉시 파이썬이 돌도록 명령어 등록을 수행합니다.

### Day 093: 시스템 전체 인프라 스트레스 테스트
*   **미션**: 한 번에 이메일 초안 생성, 요약기, RAG 검색을 동시에 실행하여 노트북이 뻗지 않고 잘 견디는지 검증합니다.

### Day 094: 100% 오프라인 완전 자동화 시연
