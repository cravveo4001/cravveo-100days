# Cravveo Channel Direction: 채널 방향성 정의서

> Cravveo는 "초보자가 AI를 파트너로 삼아 로컬 노트북과 Google Colab을 오가며 1인 기업의 작업 시스템을 직접 구축하는 과정"을 기록하는 채널입니다.

---

## 1. 채널 정체성
Cravveo Company는 완성된 전문가의 강의 채널이 아니라, 초보자가 실제로 부딪히며 배우는 과정을 공개하는 Build in Public형 기록 채널입니다.

핵심 문장은 다음과 같습니다.

> 나만의 1인 기업을 만들기 위해, 로컬 AI와 Google Colab GPU 파인튜닝, 자동화 시스템을 직접 구축하는 100일 실험.

---

## 2. 핵심 시청자
이 채널은 다음 사람을 위해 만듭니다.

1. AI를 써보고 싶지만 어디서 시작해야 할지 모르는 사람
2. 로컬 AI, Google Colab, RAG, 파인튜닝이 궁금하지만 어렵게 느끼는 사람
3. 1인 기업, 자동화, 생산성 시스템에 관심 있는 사람
4. 완성된 결과보다 실제 시행착오와 성장 과정을 보고 싶은 사람

---

## 3. 콘텐츠 원칙
영상은 아래 원칙을 지킵니다.

1. 모르는 것을 아는 척하지 않습니다.
2. 실패와 오류를 편집으로 숨기지 않습니다.
3. 명령어, 에러 메시지, 해결 과정을 화면에 남깁니다.
4. 매일의 성공 조건을 작게 정의하고 끝까지 확인합니다.
5. AI를 마법처럼 포장하지 않고, 실제 도구이자 파트너로 다룹니다.

---

## 4. 톤앤매너
말투는 쉽고 솔직하게 유지합니다.

너무 가볍게 웃기려고 하기보다, 시청자가 "나도 따라갈 수 있겠다"는 감각을 얻도록 설명합니다.

전문 용어는 필요할 때만 사용하고, 사용한 뒤에는 쉬운 비유로 다시 풀이합니다.

---

## 5. 100일 콘텐츠 축
100일 프로젝트는 아래 흐름으로 진행합니다.

1. 리눅스와 Python 개발 환경 구축
2. Ollama 기반 로컬 LLM 실행
3. AnythingLLM과 오프라인 RAG 실험
4. Google Colab GPU 환경에서 학습 데이터 제작과 LoRA 파인튜닝
5. 코랩에서 만든 가중치를 로컬 노트북으로 회수
6. GGUF 변환 및 Ollama 모델 등록
7. 옵시디언 지식 저장소와 로컬 RAG 결합
8. 이메일, 요약, 음성 브리핑 자동화
9. 1인 기업 운영 시스템으로 확장

---

## 6. 영상 기본 구조
각 영상은 아래 구조를 기본으로 합니다.

1. 오늘의 목표를 30초 안에 말합니다.
2. 현재 상태를 보여줍니다.
3. 명령어 또는 작업을 직접 실행합니다.
4. 에러가 나면 원인과 해결 과정을 기록합니다.
5. 성공 조건을 확인합니다.
6. 다음 날 할 일을 짧게 예고합니다.

---

## 7. 금지할 것
아래 방식은 피합니다.

1. 실제로 해보지 않은 내용을 성공한 것처럼 말하기
2. 시청자가 따라 칠 수 없는 빠른 진행
3. 에러 화면을 숨기고 결과만 보여주기
4. 도구 이름만 나열하고 왜 쓰는지 설명하지 않기
5. AI를 과장된 만능 해결책처럼 표현하기
