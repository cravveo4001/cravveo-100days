# 2026-05-29 Day 006 Log

## 오늘의 목표

Python 코드로 Ollama API를 호출해서 AI 답변을 받아온다.

## 실행한 명령어

```bash
cd ~/projects/cravveo_ai
source venv/bin/activate
cd ~/finetuning-project/6일차
pip install requests
python ask_ai.py
curl http://localhost:11434
ollama list
```

## 확인한 결과

- Python `requests` 라이브러리로 Ollama API 호출 성공
- `ask_ai.py` 실행 → AI 답변 출력 확인
- qwen3.5:2b 모델로 1인 기업 AI 활용 3줄 요약 답변 수신

## 막힌 지점

- `gemma4:e2b` (7.2 GB) 실행 시 컴퓨터 다운 (화면 녹화 중이라 RAM 부족)
- `source venv/bin/activate` 경로 오류 (finetuning-project에서 실행해서 venv 없음)
- `stream: False` + qwen3.5 thinking 모드로 인해 200 토큰을 전부 생각에 써버려 response 비어있는 현상

## 해결 방법

- 모델을 `qwen3.5:2b` (2.7 GB)로 교체 → RAM 안정
- venv는 `~/projects/cravveo_ai`에서 활성화 후 6일차 폴더로 이동
- `"think": False` 옵션 추가로 thinking 모드 비활성화 → 정상 답변 출력

## 성공 조건

- [x] `ollama list` 에서 모델 확인
- [x] 가상환경 활성화 완료
- [x] `pip install requests` 완료
- [x] `ask_ai.py` 작성 완료

## 오늘의 핵심 발견

Python 코드 몇 줄로 AI를 "도구처럼" 호출할 수 있다는 것을 처음 경험했다.
터미널 채팅이 아닌 코드로 AI를 제어하는 것이 자동화의 시작점임을 확인했다.
qwen3 계열 모델은 thinking 모드를 끄지 않으면 응답이 비어있을 수 있다.
