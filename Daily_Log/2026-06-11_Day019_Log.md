## 실행한 작업

- 코랩 T4 GPU 런타임 확인
- JSON vs JSONL 차이 정리 (텍스트 셀에 직접 작성)

- 크라베오 관련 JSONL 데이터 5개 직접 작성 (Alpaca 포맷)

- JSONL 파일 다시 읽기 코드 실행
- HuggingFace `datasets`로 로드 확인 (`load_dataset`)
- YouTube 업로드 문서 작성

## 확인한 결과

- Alpaca 포맷: instruction/input/output 구조 — 지시 학습에 적합
- ShareGPT 포맷: human/gpt 대화 구조 — 챗봇에 적합
- `cravveo_sample.jsonl` 파일 생성 완료 (5개 항목)
- JSONL 파일 다시 읽기 성공 — 각 줄 정상 출력
- HuggingFace datasets로 로드 확인 — `Dataset({features: ['instruction', 'input', 'output'], num_rows: 5})`

## 오늘의 핵심 발견

- AI는 그냥 텍스트 파일을 읽지 않는다. JSONL이라는 정해진 형식으로만 받아들인다
- JSONL을 모르면 데이터가 아무리 많아도 AI가 읽지 못한다
- `json.dumps()`에 `ensure_ascii=False`를 넣어야 한글이 정상 저장된다
- 파인튜닝은 결국 "이 질문에 이렇게 대답해라"를 수백 번 보여주는 과정이다
