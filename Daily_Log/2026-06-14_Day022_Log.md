# 2026-06-14 Day 022 Log

## 오늘의 목표

HuggingFace 계정에서 토큰을 발급받고 코랩에서 로그인을 완료한다. 완성된 46개 데이터셋을 HuggingFace에 업로드하기 위한 사전 준비를 마친다.

## 실행한 작업

- 코랩 노트북 처음부터 순서대로 재정리 (cravveo_finetuning.ipynb 새로 작성)
- HuggingFace Write 권한 토큰 발급
- Colab Secrets(🔑)에 토큰 안전하게 등록 (HF_TOKEN)
- `login(token=userdata.get('HF_TOKEN'))` 방식으로 로그인 코드 작성
- HuggingFace 로그인 완료 확인 (`whoami()` — 사용자명: cravveo)
- YouTube 업로드 문서 작성
- 토큰 보안 실수 공개 및 기록

## 확인한 결과

- 코랩 노트북 셀 18개 순서 정리 완료 (Section 0~4, Day 017~022 전체 포함)
- HuggingFace 로그인 성공 — 사용자명 cravveo 출력 확인
- Colab Secrets 방식으로 토큰 보안 처리 완료

## 막힌 지점

1. **token.txt 파일에 토큰 저장** — GitHub에 올라갈 뻔한 보안 실수
2. **공개 노트북에 `login(token="값")` 직접 작성** — 코드에 토큰 노출 위험
3. **LocalProtocolError: Illegal header value** — Secrets에 토큰 붙여넣을 때 줄바꿈 문자(`\r\n`) 포함

## 해결 방법

1. token.txt 즉시 삭제, 기존 토큰 HuggingFace에서 삭제 후 재발급
2. `login()` 방식 → Colab Secrets + `userdata.get('HF_TOKEN')` 방식으로 변경
3. Secrets에서 HF_TOKEN 삭제 후 토큰 다시 등록 (줄바꿈 없이 정확히 붙여넣기)

- 토큰은 파일로 저장하거나 코드에 직접 쓰면 절대 안 된다 — Colab Secrets가 정답
- 공개 노트북에서의 보안은 "당연히 괜찮겠지"가 아니라 의식적으로 챙겨야 한다
- 실수를 기록하고 공개하는 것 자체가 Build in Public의 핵심이다
