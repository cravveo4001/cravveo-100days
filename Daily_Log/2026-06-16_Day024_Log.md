# 2026-06-16 Day 024 Log

## 오늘의 목표

Unsloth로 Gemma 2B 모델을 불러오고, HuggingFace에 올린 내 데이터셋을 코드 한 줄로 연결한다. 파인튜닝 뼈대 코드를 완성해서 내일 LoRA + SFTTrainer를 바로 붙일 수 있는 상태를 만든다.

## 실행한 작업

- Unsloth `FastLanguageModel.from_pretrained()` 로 Gemma 2B 4비트 양자화 로드
- `load_dataset("cravveo/cravveo-briefing-dataset", split="train")` 데이터셋 연결
- 프롬프트 포맷 변환 함수 작성 (instruction + output → 하나의 텍스트)
- 토크나이저로 모델-데이터 연결 확인
- max_length 512 → 2048 수정 (데이터 잘림 방지)
- YouTube 업로드 문서 작성

## 확인한 결과

- Gemma 2B 모델 로드 성공
- 데이터셋 42~53개 로드 확인 (briefing + identity 합산)
- 프롬프트 포맷 변환 완료 (`text` 컬럼 생성)
- 토크나이저 처리 완료 — 2048 토큰 기준 정상 작동
- 파인튜닝 뼈대 코드 완성

## 막힌 지점

1. **NameError: 'ample' is not defined** — `sample` 변수명에서 `s` 누락 (어제 `ataset`에 이어 두 번째 같은 유형 오타)
2. **토큰 수 512개** — `max_length=512`로 인해 데이터가 잘리고 있었음

## 해결 방법

1. `ample` → `sample` 수정 후 재실행
2. `max_length=512` → `max_length=2048` 로 변경 (모델 `max_seq_length`와 일치)

## 성공 조건

- [x] Unsloth FastLanguageModel 모델 로드 성공
- [x] `load_dataset()` 한 줄로 HuggingFace 데이터 로드
- [x] 프롬프트 포맷 변환 완료
- [x] 토크나이저 연결 확인 (2048 토큰)
- [x] YouTube 업로드 문서 작성 완료
- [ ] 영상 녹화 (사용자 진행)

## 오늘의 핵심 발견

- 토큰 수가 정확히 `max_length`와 같으면 데이터가 잘리고 있는 것이다
- 토크나이저 `max_length`는 반드시 모델 `max_seq_length`와 맞춰야 한다
- 데이터가 모델 안으로 처음 들어간 날 — 파인튜닝 파이프라인의 골격이 완성됐다

## 오늘 솔직한 후기

어제 `ataset` 오타를 낸 지 하루 만에 `ample` 오타를 또 냈다. 붙여넣을 때 맨 앞 글자가 잘리는 패턴인데 습관적으로 확인을 안 한다. 그래도 모델과 데이터가 처음으로 연결되는 걸 확인했다. 내일 LoRA를 붙이면 실제 학습이 시작된다.
