## 막힌 지점

1. **RuntimeError: CUDA SETUP ERROR: Missing dependency: libnvJitLink.so.13**
   - `!pip install bitsandbytes --upgrade` → CUDA 경로 미등록 오류
   - `bitsandbytes==0.43.1` → unsloth 2026.6.7이 >=0.45.5 요구
   - `bitsandbytes==0.45.5` → CUDA 13.0 바이너리 없음, CPU 폴백도 broken
   - ldconfig 등록 → 효과 없음
   - symlink 생성 → 효과 없음
   - `ctypes.CDLL()` (RTLD_LOCAL 기본값) → 효과 없음
   - `os.environ["LD_LIBRARY_PATH"]` → 효과 없음

2. **ctypes 셀 위치 오류**
   - `import unsloth` 앞에 배치 → 효과 없음
   - 런타임 재시작 이후, `FastLanguageModel.from_pretrained()` 직전에 배치 → 성공

3. **PicklingError** (학습 후)
   - `Can't pickle trl.trainer.sft_config.SFTConfig` — 학습 완료 후 메트릭 저장 단계에서 발생
   - 학습 자체는 완료됐고 체크포인트도 저장됨 — 무시 가능

## 해결 방법

1. **bitsandbytes CUDA 오류**: 런타임 연결 해제 및 삭제 → 새 VM 연결 (CUDA 환경 초기화)
2. **libnvJitLink.so.13 미등록**: ctypes RTLD_GLOBAL 방식으로 해결
   ```python
   import os, ctypes
   os.environ["LD_LIBRARY_PATH"] = "/usr/local/lib/python3.12/dist-packages/nvidia/cu13/lib:" + os.environ.get("LD_LIBRARY_PATH", "")
   ctypes.CDLL("/usr/local/lib/python3.12/dist-packages/nvidia/cu13/lib/libnvJitLink.so.13", mode=ctypes.RTLD_GLOBAL)
   ```
3. **셀 위치**: 런타임 재시작 이후, FastLanguageModel 로드 직전에 배치

## 오늘의 핵심 발견

- `pip install`은 디스크에만 설치된다. 런타임 재시작이 필요한 이유는 메모리 초기화 때문이다
- `ctypes.CDLL()` 기본 모드는 `RTLD_LOCAL` — 이 상태로는 다른 .so 파일에서 심볼을 못 찾는다. `mode=ctypes.RTLD_GLOBAL`이어야 전역 등록된다
- 라이브러리 파일이 존재해도 동적 링커 경로에 등록되지 않으면 `No such file or directory`가 뜬다
- ctypes 셀의 위치가 중요하다: 런타임 재시작이 메모리를 초기화하므로 재시작 이후에 다시 실행해야 한다

## 오늘 솔직한 후기

LoRA + SFTTrainer 코드 자체는 10분이면 됐다. 그런데 bitsandbytes CUDA 오류 하나가 하루를 잡아먹었다. libnvJitLink.so.13 파일이 시스템에 분명히 있는데 못 찾는 황당한 상황이었다. 버전 바꾸고, 경로 설정하고, symlink 걸고, 런타임 재시작하고... 결국 ctypes RTLD_GLOBAL + 셀 위치 조정으로 해결됐다. 중간에 Claude가 셀 위치를 잘못 안내해서 돌아간 부분도 있었다. 그래도 loss가 처음으로 줄어드는 걸 봤다. 25.5에서 15.4. 숫자가 내려가는 게 이렇게 뿌듯할 줄 몰랐다.
