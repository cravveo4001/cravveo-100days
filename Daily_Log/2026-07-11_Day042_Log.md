- RAG 없는 워크스페이스에서 첫 질문("크라베오 컴퍼니가 뭐야?") 테스트 → 즉시 심각한 오류 발견 (가짜 URL `qna.cravve.io`, `cravvbot.com`, 없는 서비스 나열)
- 원인 진단: 정체성 데이터 20개가 Obsidian 데이터 203개에 비해 9%밖에 안 돼서 학습 신호가 묻힘 (과소적합)
- 수정 시도: 정체성 데이터 20개를 10배 복제(200개)해서 Obsidian과 비율을 맞추고 `Day042_Retrain.ipynb`로 재학습 → `cravveo:v6` 등록
- v6 재테스트 → 더 심각하게 붕괴 (한국어/영어에 아랍어·힌디어·태국어·베트남어 글자가 무작위로 섞여 나옴)

## 확인한 결과

- **v5**: 정체성 데이터 부족 → 과소적합 → 완전히 지어낸 내용으로 채움

## 오늘의 핵심 발견

작은 데이터셋으로 파인튜닝할 때 "비율을 맞추면 되겠지"라는 생각이 얼마나 순진했는지 깨달은 하루. 9%는 너무 적어서 실패했고, 단순 복제로 비율을 확 올렸더니 이번엔 반복이 너무 심해서 다른 방식으로 실패했다. 두 실패의 원인이 서로 다르다는 걸(과소적합 vs 훈련 불안정) 구분해낸 게 오늘의 진짜 수확. "이번엔 되겠지"로 세 번째 재학습을 또 시도하지 않고, 두 번 실패한 시점에서 멈추고 원인을 정리한 것도 Day 040부터 지켜온 원칙(안 되는 걸 계속 붙잡지 않기)을 실제로 지킨 사례.
