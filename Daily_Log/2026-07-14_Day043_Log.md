# 2026-07-14 Day 043 Log

## 오늘의 목표

Day 042에서 만든 정체성 평가세트(30문항)로 `cravveo:v4`를 처음부터 끝까지 테스트해서 기준선(baseline)을 만든다.

## 실행한 작업

- AnythingLLM "default" 워크스페이스에서 테스트 시작 → 채팅 모드가 "에이전트"로 되어있는 걸 발견 (Day 038과 같은 버그가 다른 워크스페이스에 남아있었음)
- Ollama가 요청에 응답 못 하는 오류 발생 → 서버는 정상 작동 중이었고, CPU만으로 500토큰 답변 생성에 1분 이상 걸려 타임아웃난 것으로 확인
- 터미널 `ollama run cravveo:v4`로 전환, `/set parameter num_predict 150`으로 임시로 답변 길이를 줄여 테스트 속도 개선
- `43일차/v4_테스트.md`에 30문항(A1~A20, B1~B10) 전체 질문/답변 기록
- 전체 채점 후 `정체성_평가세트.md` 결과표에 기입

## 확인한 결과

- **v4 채점: O 0개 / △ 2개(A4, B1) / X 28개 (30문항 중)**
- v4는 Day 041 데이터 정리 **이전**(오염된 92개 데이터)으로 학습된 버전이라, 정체성 질문에 거의 전부 실패함. 지금까지 "v4는 안정적"이라고 여겨온 건 날짜 질문 위주로만 테스트했기 때문에 몰랐던 착각이었음
- A5~A20 대부분이 질문 내용과 무관하게 "Olllama가 뭐야?" / "YouTube Channel: Day-by-Day 파인튜닝... separate information (업체.info)..." 같은 고정 문장으로 수렴함 — 단순 오답이 아니라 특정 텍스트 패턴에 붕괴된 상태로 보임
- 결론: 현재 존재하는 모든 버전(v2, v4=구 오염 데이터, v5=과소적합, v6=훈련 불안정)이 이 평가세트를 통과하지 못함. "쓸만한 기준선"이 사실 없었던 상태

## 막힌 지점

- AnythingLLM에서 Ollama 응답 타임아웃 (서버 다운 아님, CPU 생성 속도 문제)
- "default" 워크스페이스의 채팅 모드가 에이전트로 설정되어 있던 것 (Day 038 버그 재발)

## 해결 방법

- 타임아웃: 터미널 `ollama run`으로 전환해 우회, `num_predict`를 세션 한정으로 150으로 낮춰 테스트 속도 개선
- 에이전트 모드 문제: 이번엔 별도로 안 고치고 터미널 테스트로 우회 (다음에 이 워크스페이스 쓸 일 있으면 고치기)

## 오늘의 핵심 발견

"v4는 그나마 안정적이다"라는 그동안의 믿음이 즉흥적 테스트(질문 한두 개)에 기반한 착각이었다는 걸, 평가세트로 제대로 측정하고서야 알게 됐다. 감으로 판단했으면 계속 v4를 신뢰하며 그 위에 뭔가를 쌓았을 텐데, 고정된 30문항으로 체계적으로 재보니 v2/v4/v5/v6 네 버전 다 정체성 질문에 사실상 실패 상태였다. 안 좋은 소식이지만, 이게 정확히 어제 정한 방법론("즉흥 판단 대신 고정 평가세트")이 의도한 대로 작동한 사례다 — 착각을 데이터로 깨는 것.
