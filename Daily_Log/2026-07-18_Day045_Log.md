## 실행한 작업

- **급한 버그 발견**: Day 044에 베이스 모델을 3B→8B로 바꿨는데, 정체성 데이터 중 "지금 쓰는 모델이 뭐야?" 관련 4개 항목이 여전히 "Llama 3.2 3B Instruct"라고 되어있는 걸 발견 → "Llama 3.1 8B Instruct"로 수정, HuggingFace 재업로드
- 계속 틀리던 항목(Chrome 확장 개수/목록, ablescan.app, cravveo.com, 하루 일과) 보강 — 표현 14개 추가(`45일차/cravveo_identity_reinforce.jsonl`), 특히 ablescan.app과 cravveo.com을 헷갈려하던 문제를 직접 겨냥한 "둘의 차이가 뭐야?" 항목도 추가
- 정체성 데이터 80개 → 94개로 확장, HuggingFace 재업로드
- Confabulation(지어내기) 원인 조사: 학습 데이터 답변 길이를 실측(평균 72자)하고, 현재 `num_predict`(120~500) 설정과 비교
- **당초 계획을 바꿔 같은 날 v8 재학습까지 진행** (페이스 원칙상 원래는 다음 주 예정이었으나 직접 판단으로 진행): 44일차 노트북 재사용, cell 4가 HuggingFace의 최신 94개 데이터를 자동으로 가져와 학습
- 학습 중 데이터 업로드 실수 발견 및 수정: cell 4가 obsidian 데이터셋 파일을 요청했는데 실수로 `cravveo_identity_94.jsonl`을 업로드해서 "정체성 94 + 가짜 obsidian 94 = 188개(50:50)"로 조합될 뻔함 → cell 4로 돌아가 올바른 `44일차/obsidian_dataset_cleaned.jsonl`로 재업로드 지시
- `정체성_평가세트.py` 신규 작성 (모델명을 인자로 받는 범용 자동 평가 스크립트, 키워드 기반 1차 채점 포함)
- v8을 Ollama에 등록(`cravveo:v8`, num_predict 500→100으로 조정) 후 `정체성_평가세트.py`로 30문항 자동 실행, 이어서 30개 답변 전부 수동 재채점

## 확인한 결과

- 모델 버전 오류 4건 수정 완료, 정체성 데이터 94개로 확장 완료 (HuggingFace 반영 확인됨 — "3B/3.2" 언급 0건, "8B/3.1" 언급 4건)
- **Confabulation 원인 추정**: 학습 데이터 답변은 평균 72자(짧음)인데, 실제 답변 생성 시 `num_predict`가 120~500으로 훨씬 길게 설정되어 있음. 모델이 학습한 만큼 짧게 답하고 나면, 남은 토큰 예산을 채우려고 계속 이어 말하다가 지어내는 것으로 추정됨
- **v8 자동채점: O12/△10/X8** (v7 대비 개선처럼 보임) — 그러나 **수동 재채점 결과 O5/△5/X20으로, v7(O8/△12/X10)보다 오히려 저하**. 자동채점은 키워드 매칭 방식이라 "불가능"에 "가능"이 포함되어 오탐하는 등 실제보다 후하게 나온 것으로 확인됨 (`정체성_평가세트_cravveo_v8_결과.md` 참고)
- num_predict를 500→100으로 낮췄음에도 confabulation은 줄지 않음 (ablescan.app, cravveo.com, 모델 크기 등에서 여전히 지어낸 내용 다수)
- **원인 불확실**: LoRA는 오늘 10:37(KST) 새로 커밋됐고 HuggingFace 데이터셋과 보강 항목(ablescan.app/cravveo.com 7개) 내용도 정상 확인됨. 다만 Colab 세션이 끊겨서 "잘못된 obsidian 파일 업로드를 재업로드로 제대로 고쳤는지"를 보여주는 cell 4 최종 로그를 확인할 수 없어, (1) 파일 재업로드가 실제로 반영 안 됐을 가능성과 (2) 특정 사실(ablescan.app 등)의 학습 예문이 전체 297개 중 7개뿐이라 너무 희석됐을 가능성 둘 다 배제 못함

## 막힌 지점

Colab 세션 연결이 끊겨서 실제 학습에 사용된 데이터 조합을 사후 검증할 방법이 없었음 — cell 4 출력 로그를 캡처/저장해두지 않은 것이 원인.

## 오늘의 핵심 발견

베이스 모델을 8B로 바꿔놓고 정작 학습 데이터엔 "3B"라고 남겨뒀던 걸 오늘에서야 발견했다 — 어제 성과에 들떠서 세부 사항을 놓칠 뻔했다. 또한 "왜 자꾸 지어내지?"를 감으로 넘기지 않고 실제로 답변 길이를 재봤더니, 학습 데이터(72자)와 생성 설정(120~500토큰)의 불일치라는 구체적 원인이 나왔다.

그런데 같은 날 바로 v8을 재학습해봤더니 자동채점(O12/△10/X8)은 좋아 보였지만, 30개 답변을 직접 다시 읽어보니 실제로는 v7보다 나빠졌다(O5/△5/X20). 자동채점의 허점("불가능"에 "가능"이 포함돼 오탐)도 확인했고, 무엇보다 재학습에 실제로 어떤 데이터가 들어갔는지 사후에 검증할 방법이 없다는 구조적 허점도 발견했다. "실행됐다"와 "제대로 됐다"는 다르다는 걸, 그리고 검증 가능성 자체를 학습 과정에 미리 심어둬야 한다는 걸 배운 하루.
