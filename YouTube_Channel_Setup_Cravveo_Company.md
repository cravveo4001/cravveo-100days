# YouTube Channel Setup: Cravveo Company

> 목적: `Cravveo Company` 유튜브 채널을 새로 만들 때 빠짐없이 설정하기 위한 체크리스트입니다.

---

## 1. 채널 기본 결정

채널명:

```text
Cravveo Company
```

추천 핸들:

```text
@CravveoCompany
```

대체 핸들:

```text
@cravveo
@cravveo_ai
@cravveo_company
@CravveoAI
```

채널 포지션:

```text
초보자가 로컬 AI, Google Colab 파인튜닝, RAG, 옵시디언, 자동화를 배워가며
1인 기업용 AI 시스템을 직접 구축하는 100일 기록 채널.
```

---

## 2. 채널 생성 방식

`Cravveo Company`는 개인 이름이 아니라 프로젝트/브랜드 이름이므로 **브랜드 채널 방식**을 추천합니다.

채널 생성 순서:

1. YouTube에 로그인합니다.
2. 프로필 아이콘을 누릅니다.
3. 설정 또는 채널 목록으로 이동합니다.
4. 새 채널 만들기를 선택합니다.
5. 이름에 `Cravveo Company`를 입력합니다.
6. 핸들을 설정합니다.
7. 채널을 생성합니다.

주의:

- 개인 실명 채널보다 브랜드 채널이 장기 운영에 유리합니다.
- 나중에 운영자를 추가할 가능성이 있으면 브랜드 계정 구조가 더 좋습니다.
- 채널 생성 직후에는 핸들 URL을 반드시 확인합니다.

## 3. 프로필 설정

YouTube Studio에서 이동:

```text
Customization -> Profile
```

설정할 항목:

1. 프로필 사진
2. 배너 이미지
3. 동영상 워터마크
4. 채널 이름
5. 핸들
6. 채널 설명
7. 외부 링크

프로필 사진 콘셉트:

```text
단순한 C 또는 Cravveo 로고.
작은 원형 아이콘에서도 알아볼 수 있게 글자와 대비를 크게 한다.
```

배너 이미지 문구:

```text
Cravveo Company
100 Days of Local AI, Fine-Tuning, RAG and Automation
```

한국어 배너 문구 대안:

```text
Cravveo Company
초보자의 100일 로컬 AI 구축기
```

워터마크:

```text
Cravveo
```

또는 작은 `C` 로고.

## 6. 국가/지역 설정

YouTube Studio에서 이동:

```text
Settings -> Channel -> Basic info -> Country of residence
```

추천:

```text
South Korea
```

이 설정은 YouTube Partner Program 자격과도 연결될 수 있으므로 실제 거주/운영 국가에 맞춥니다.

## 7. 시청자층 설정

YouTube Studio에서 이동:

```text
Settings -> Channel -> Advanced settings -> Audience
```

추천:

```text
No, set this channel as not made for kids.
```

이유:

- 채널 주제가 개발, AI, 터미널, 파인튜닝, 자동화입니다.
- 어린이 전용 콘텐츠가 아닙니다.

주의:

- 실제 영상이 어린이 대상이면 영상별로 별도 설정해야 합니다.
- 설정은 법적 책임이 걸릴 수 있으므로 콘텐츠 성격에 맞게 신중히 선택합니다.

## 10. 첫 영상 업로드 설정

```text
AI 1인 기업 만들기 Day 001 | 리눅스에 Python과 Git 설치부터 시작합니다
```

설명:

```text
Cravveo Company 100일 프로젝트 Day 001.

오늘은 로컬 AI와 Google Colab 파인튜닝을 연결하기 위한 첫 번째 준비 단계로,
리눅스 노트북에 기본 개발 환경을 세팅했습니다.

- sudo apt upgrade

- pip 24.0
```

## 13. 운영 원칙

1. Day 번호를 제목에 반드시 넣습니다.
2. 영상 설명에는 오늘 한 일과 다음 할 일을 넣습니다.
3. 실패 장면을 숨기지 않습니다.
4. 썸네일은 단순하게 유지합니다.
5. 채널 초기에는 조회수보다 기록의 일관성을 우선합니다.
6. 모든 영상은 나중에 옵시디언/RAG 지식망으로 회수할 수 있게 정리합니다.
