**v4 합계 (2026-07-14 측정)**: O 0개 / △ 2개(A4, B1) / X 28개 — 30문항 중 사실상 전부 실패.

**패턴(v4)**: A5~A20 대부분이 질문과 무관하게 "Olllama가 뭐야?" / "YouTube Channel: Day-by-Day 파인튜닝... separate information (업체.info)..." 같은 고정 문장으로 수렴함 — 단순 오답을 넘어 특정 텍스트 패턴에 붕괴된 상태로 보임. v4는 Day 041 데이터 정리 **이전**(오염된 92개 데이터)으로 학습된 버전이라, 지금까지 "안정적"이라고 여겼던 게 착각이었음이 드러남.

**v5/v6 (2026-07-11 측정)**: 둘 다 정체성 데이터 비율 조정 실패로 폐기 (v5=과소적합/지어낸 URL, v6=10배 복제로 인한 훈련 불안정/다국어 붕괴). 자세한 내용은 Day 042 로그 참고.

**v7 합계 (2026-07-17 측정, `cravveo:v7`, Llama 3.1 8B + 정체성 80개 + Obsidian 203개 정리본)**: **O 8개 / △ 12개 / X 10개** — O+△ 20/30(67%)로 v4 대비 압도적으로 개선됨.

**패턴(v7)**:
- 서사/개념형 질문(운영자, Build in Public, 목표, LoRA 정의, 가장 힘들었던 것, 누가 운영하는지)은 대부분 O — 잘 학습됨
- **구체적 제품/도구 이름**(ablescan.app 설명, 웹 접근성 도구 이름, 정확한 모델 버전, 하루 일과)은 계속 X — 약점으로 확정
- v3~v6에서 봤던 "완전 붕괴"(무한반복, 다국어 뒤섞임)는 거의 사라졌지만, 한자/베트남어 글자가 산발적으로 한두 개씩 섞이는 잔여 현상은 남아있음 (예: "목標", "매日", "문서集合", "đó겠어")
- 학습 데이터에 없는 이야기를 자신 있게 지어내는 경향 여전함 (예: "GTX1660Ti 있다"는 거짓말, "Day 365 넘었다", 가짜 모델 버전들, 가짜 하루 스케줄)

**결론**: v7이 처음으로 "쓸만한 수준"에 도달함. 다음 개선 우선순위는 (1) 구체적 제품/도구 정보 강화, (2) 자신감 있게 지어내는 습관 억제.

**v8 합계 (2026-07-18 측정, `cravveo:v8`, Llama 3.1 8B + 정체성 94개(3B→8B 오류 수정 + 보강 14개) + Obsidian 203개, num_predict 500→100)**: **O 5개 / △ 5개 / X 20개** — **v7(O8/△12/X10)보다 오히려 저하**. 자동채점(정체성_평가세트.py)은 O12/△10/X8로 나왔으나 키워드 매칭 오탐("불가능"에 "가능" 포함 등)이 많아 신뢰 불가, 수동 재채점이 정식 기록임. 상세 답변/코멘트는 `정체성_평가세트_cravveo_v8_결과.md` 참고.

**패턴(v8)**:
- ablescan.app, cravveo.com 등 오늘 직접 보강한 항목까지 여전히 완전히 지어낸 내용으로 답함 (예: ablescan.app을 "유튜브 자막 서비스"라고 함) — 데이터 보강이 기대만큼 반영되지 않음
- 모델 버전을 "Llama 3.2"·"1B"로 반복 오답 — 오늘 고친 3B→8B 수정이 실제 학습에 반영됐는지 불확실 (HuggingFace 데이터셋 자체는 정상 확인됨)
- num_predict 500→100 인하에도 confabulation 감소 효과 뚜렷하지 않음 — 원인은 답변 길이보다 다른 요인일 가능성
- Colab 세션이 끊겨 실제 학습에 쓰인 데이터 조합(잘못된 파일 재업로드가 반영됐는지)을 사후 검증 못함 — **재발 방지: 다음 재학습부터 cell 4 출력은 cell 5 실행 전 반드시 캡처/저장**

**결론**: v8은 폐기, v7을 당분간 기준 모델로 유지. 다음 재학습(v9) 전 확인할 것: (1) cell 4 로그 캡처, (2) 특정 사실 하나당 학습 예문 수가 너무 적은 건 아닌지(예: ablescan.app 관련 7개/297개), (3) 재학습을 데이터 수정 당일에 몰아서 하지 않기.

**v9 합계 (2026-07-20 측정, `cravveo:v9`, v8과 동일한 데이터/설정으로 재학습 — cell4 로그로 데이터 조합 정상 확인됨)**: **O 3개 / △ 9개 / X 18개** — 자동채점은 O17/△10/X3으로 v7보다도 좋아 보였으나, 수동 재채점 결과 v8(O5/△5/X20)과 비슷하거나 약간 더 나쁜 수준. 상세 내용은 `정체성_평가세트_cravveo_v9_결과.md` 참고.

**패턴(v9)**:
- **데이터 조합이 정상임을 사전에 확인했는데도 v8과 비슷하게 나쁨** — 즉 Day 045의 저하가 "파일 업로드 실수" 때문만은 아니었다는 뜻
- 없는 사이트/제품을 지어내는 경향이 v8보다 더 다양해짐 (ollama-analysis.xyz, huggingface-cravveo, toolesbros.kr, TooLama, crabamd.xyz, qwen3:5b 등)
- LoRA 설명에 존재하지 않는 수식(`Loss = L(d) + λΩ(g;θg)`)을 지어내는 등 "그럴듯하지만 틀린" confabulation이 더 정교해짐
- ablescan.app을 처음으로 정확히 답함(B6: EU/WCAG 언급) — 완전히 실패하진 않았지만 A14에서는 여전히 틀림(같은 사실을 물어보는 방식에 따라 결과가 들쭉날쭉함)
- 한자/영어/힌디어 등 외국어 혼입은 여전히 산발적으로 남아있음

**결론**: v9도 폐기, v7을 계속 기준 모델로 유지. v7(정체성 80개)과 v8/v9(정체성 94개)의 공통된 차이가 정체성 데이터를 80→94개로 늘린 것이므로, 다음 조사 우선순위는 **늘어난 14개 항목(또는 94개 전체 조합)이 오히려 불안정성을 키웠는지 확인하는 것**. Day 042에서 이미 확인한 "소규모 데이터셋은 비율/내용 변화에 매우 민감하다"는 교훈과 일치함 — 다음엔 한 번에 여러 항목을 바꾸지 말고, 80개 기준에서 아주 소수(1~2개)만 바꿔가며 점진적으로 테스트하는 것을 고려.
